# Security Policy

We take the security of our software seriously. This is the default security
policy for repositories owned by **@vinersar31**. Individual repositories may
override this file with their own `SECURITY.md`.

## Supported Versions

Unless a repository states otherwise, security updates are provided for the
latest released version on the default branch.

| Version        | Supported          |
| -------------- | ------------------ |
| Latest release | :white_check_mark: |
| Older releases | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Instead, report them privately using one of the following channels:

1. **GitHub Private Vulnerability Reporting** (preferred): open the repository's
   **Security** tab and click **Report a vulnerability**. See
   [Privately reporting a security vulnerability](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
   for details.
2. **Email**: send details to **danvinersar98@gmail.com**.

Please include as much of the following as possible to help us triage quickly:

- A description of the vulnerability and its impact.
- Steps to reproduce or a proof of concept.
- Affected version(s), component(s), and configuration.
- Any suggested mitigation or fix.

## What to Expect

- **Acknowledgement**: we aim to acknowledge your report within **3 business
  days**.
- **Assessment**: we will investigate and keep you informed of our progress.
- **Resolution**: once validated, we will work on a fix and coordinate a release
  and disclosure timeline with you.
- **Credit**: with your permission, we are happy to credit you for the
  responsible disclosure.

## Disclosure Policy

We follow a coordinated disclosure process. Please give us a reasonable amount of
time to address the issue before any public disclosure. We ask that you avoid
accessing or modifying other users' data and refrain from actions that could
degrade the service while researching.

Thank you for helping keep our projects and their users safe.
